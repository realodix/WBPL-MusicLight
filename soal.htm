Case

 

Music Light

 

Music Light is a famous musical instrument shop in the town. It sells a wide range of musical instruments from the common to the rare. As time passed, the owner of this store realized how important the internet today and decided to develop a website that can ease customer to make transactions in Music Light. This website will be used by the customer/member to order musical instruments via online and promote Music Lights product to non-member visitor.

As a web developer, you are asked to develop this website in accordance with the following details:

 

GENERAL STRUCTURE

The website structure must contain these following:

         Music Light logo as the header.

         Navigation menus.

         Content of each menu

         Footer that contains copyright.

 

ROLE

The following are access granted for each role:

         Guest / Non-member Visitor

o   Home

o   Product

o   Register

o   Testimony

 

         Member

o   Home

o   Product

o   Profile

o   Cart

o   Testimony

o   Transaction

o   Logout

 

         Admin

o   Home

o   Product

o   Profile

o   Insert Product

o   Member

o   Testimony

o   Manage Transaction

o   Logout

 

Further details about those pages will be explained below this:

1.      Home page (index.php)

Halaman ini dapat diakses oleh Guest, Member, and Admin. Halaman ini menampilkan informasi umum tentang Music Light, nama pengguna, dan tanggal dengan format dd-mm-yyyy. Untuk tamu/non-member visitor, akan ada login form sehingga pengguna dapat login ke situs web. Menunjukkan pesan kesalahan ketika proses login gagal. Tabel di bawah menunjukkan Daftar validasi untuk login form:

Field

Validation

Username

Must be filled.

Password

Must be filled.

Username & Password

Match with username and password in the database.

Gunakan teknik MD5 untuk encrypt password. Berikan pilahan remember me untuk mengatur cookie ke web browser. Jika remember me di cheklist, cookie akan diatur ke web browser tetapi jika itu tidak dicentang, cookie tidak akan di set. Juga memvalidasi Username textbox akan secara otomatis diisi setiap kali pengguna mengakses situs ini lagi dalam waktu 60 menit karena kuki ditetapkan.

 

2.      Registration page (registration.php)

Halaman ini digunakan Guest untuk membuat account baru dan menjadi anggota Music Light. Halaman pendaftaran hanya dapat diakses apabila pengguna dalam keadaan tidak login. Tabel di bawah menunjukkan daftar validasi untuk halaman pendaftaran:

Field

Validation

Name

Must be filled and contain only alphabet letters.

Username

Must be filled and cannot take an existing username.

Password

Must be filled, contain numbers and alphabet letters, and more than 5 characters.

Confirm Password

Must be filled and match with password.

Gender

Must be chosen.

Address

Must be filled and contain street word.

Phone

Must be filled and contain only numbers.

Email

Must be filled and match with email format.

Example:

admin@admin.com correct

abc@ad@.com wrong

 

3.      Product page (product.php)

Halaman ini dapat diakses oleh Guest, Member, and Admin. Halaman ini menampilkan setiap produk yang tersedia yang dijual olehMusic Light. Setiap Product harus menampilkan Image, Product ID, Brand, Instrument Type, Price, and Stock. Berikan fitur paging and searching pada halaman ini. Setiap halaman berisi hanya 3 produk dan semua pengunjung dapat mencari berdasarkan Brand, Instrument Type, atau Price.

Important information for this page is:

         Guest hanya dapat melihat produk yang memiliki Stock lebih dari 0.

         Member dapat melihat produk yang memiliki Stock lebih dari 0 dan dapat menambahkan produk ke keranjang nya. Tabel di bawah menunjukkan daftar validasi untuk anggota ketika menambahkan produk ke keranjang mereka:

Field

Validation

Quantity

Must be filled, contain only numbers, and must not bigger than the available stock.

Update quantity if the member buys the same product.

         Admin can view all, update, and delete the products.

 

4.      Insert product page (insertproduct.php)

Halaman ini hanya dapat diakses oleh Admin. Halaman ini digunakan oleh Admin untuk memasukkan produk baru. Informasi yang harus disimpan ketika memasukkan produk baru adalah Brand, Type instrumen, harga, stock, dan gambar. Anda diminta untuk menambahkan fitur upload file di Halaman ini untuk meng-upload gambar produk. Tabel di bawah menunjukkan daftar validasi untuk memasukkan Produk:

Field

Validation

Brand

Must be filled and more than 3 characters.

Instrument Type

Must be chosen.

Price

Must be filled and contain only numbers and bigger than 500000.

Stock

Must be filled and contain only numbers and bigger than 10.

Image

Must be filled and chosen from a file with .jpg extension.

 

5.      Update product page (updateproduct.php)

Halaman ini hanya dapat diakses oleh Admin melalui halaman produk. Halaman ini digunakan oleh Admin untuk memperbarui produk yang ada. Informasi yang diperlukan untuk memperbarui Product adalah Brand, Instrument Type, Price, Stock, and Image. Jika Admin tidak meng-upload gambar baru, itu berarti produk masih menggunakan gambar lama. Validasi dalam Halaman ini sama seperti Insert Product.

 

6.      Profile page (profile.php)

Halaman ini hanya dapat diakses oleh Member or Admin. Halaman ini menampilkan Username, Name, Gender, Address, Cell Phone, and Email dari user. Pada Halaman ini, pengguna dapat mengubah profil. Tabel di bawah menunjukkan Daftar validasi untuk Ubah profil:

Field

Validation

Name

Must be filled and contain only alphabet letters.

Address

Must be filled and contain street word.

Must contain 2 words or more.

Phone

Must be filled and contain only numbers.

Email

Must be filled and match with email format.

Example:

admin@admin.com correct

abc@ad@.com wrong

Di bawah ini adalah perubahan bentuk profil, akan ada perubahan bentuk password bagi pengguna untuk mengubah password mereka. Tabel di bawah menunjukkan daftar validasi untuk Ubah sandi:

Field

Validation

Old Password

Must be filled and match the users password in database.

New Password

Must be filled and contain numbers and alphabet letters.

Confirm Password

Must be filled and match New Password.

 

7.      Member page (member.php)

Halaman ini hanya dapat diakses oleh Admin. Hal ini digunakan untuk melihat semua informasi anggota kecuali password. Selain itu, Admin juga dapat menghapus anggota dengan mengklik Hapus kecuali dirinya. Sediakan fitur paging di Halaman ini, setiap halaman menampilkan hanya 5 anggota.

 

8.      Cart page (cart.php)

Halaman ini hanya dapat di akses oleh Member. Fungsi dari Halaman ini adalah untuk menampilkan setiap produk yang telah ditambahkan oleh anggota dari halaman produk. Informasi yang harus ditampilkan dalam setiap produk yang dipilih Image, Product ID, Brand, Instrument Type, Price, Quantity, and Sub Total (Price * Quantity). Setelah menambah Total Price dari semua produk. Setiap produk akan memiliki tombol Delete untuk menghapus atau membatalkan produk tersebut. Terakhir, tambahkan tombol Buy untuk menambahkan semua informasi yang tercantum di keranjang ke Transaction page dan kosongkan halaman keranjang. Member harus menunggu hingga Admin approve transaksinya.

 

9.      Transaction page (transaction.php)

Halaman ini hanya dapat diakses ole Member and Admin. Halaman ini menampilkan Transaction ID, Username, Date, and Status dari transaksi. Tambahkan link untuk detail.php di setiap Transaction ID, sehingga pengguna dapat melihat rincian tentang transaksi dipilih. Member hanya dapat melihat his/her transactions. Admin dapat melihat semua transactions yang dibuat oleh dan dapat menghapusnya and approve each transaction. If Admin approves a transaction, validate if the Stock of the product is still bigger than or equal to the Quantity of product in transaction page. If the transaction is valid (Stock >= Quantity), do not forget to reduce Stock (as many as Quantity), and update the Date so it becomes the date when Admin approve the transaction. But if the transaction is not valid (Stock < Quantity) shows error message that tells users about the insufficient stock.

 

10.  Detail page (detail.php)

This page can only be accessed by Member and Admin through Transaction page. The function of this page is to display the detail about the transaction. This page only shows Image, Product ID, Brand, Instrument Type, Price, Quantity, Sub Total and Total Price of all products. Validate the Member can only view his/her transactions, while Admin can view all Members transactions.

 

11.  Testimony page (testimony.php)

This page can be accessed by Guest, Member, and Admin. It displays the testimony from Member and Admin. For Guest, he/she can only view the testimonies. For Member, he/she can view the testimonies and write (add) testimony for Music Light but it will be shown after Admin approve his/her testimony. For Admin, he/she can approve and delete the testimony from Member. Admin can also write testimony directly without someone else permission.

Field

Validation

Testimony

Must be filled.

Must be between 5 and 160 characters.

Must not contain HTML tags.

 

12.  Logout page (logout.php)

This page can only be accessed by logged in user to redirect him/her back to the home page for guest/non-member visitor and clear all sessions that have been stored.
