Soal

Case

 

Music Light

 

Music Light is a famous musical instrument shop in the town. It sells a wide range of musical instruments from the common to the rare. As time passed, the owner of this store realized how important the internet today and decided to develop a website that can ease customer to make transactions in Music Light. This website will be used by the customer/member to order musical instruments via online and promote Music Lights product to non-member visitor.

As a web developer, you are asked to develop this website in accordance with the following details:

 

GENERAL STRUCTURE

The website structure must contain these following:

         Music Light logo as the header.

         Navigation menus.

         Content of each menu

         Footer that contains copyright.

 

ROLE

The following are access granted for each role:

         Guest / Non-member Visitor

o   Home

o   Product

o   Register

o   Testimony

 

         Member

o   Home

o   Product

o   Profile

o   Cart

o   Testimony

o   Transaction

o   Logout

 

         Admin

o   Home

o   Product

o   Profile

o   Insert Product

o   Member

o   Testimony

o   Manage Transaction

o   Logout

 

Further details about those pages will be explained below this:

1.      Home page (index.php)

This page can be accessed by Guest, Member, and Admin. This page displays general information about Music Light, username of logged in user, and todays date with format dd-mm-yyyy. For guest/non-member visitor, there will be a login form so users can login to the website. Shows the error message when the login process is failed. The table below shows the list of validation for the login form:

Field

Validation

Username

Must be filled.

Password

Must be filled.

Username & Password

Match with username and password in the database.

Use MD5 technique to encrypt the password. Also give remember me option to set cookie into web browser. If remember me is checked, the cookie will be set into web browser but if it is not checked, the cookie will not be set. Also validate Username textbox will be automatically filled every time users access this website again within 60 minutes since the cookie is set.

 

2.      Registration page (registration.php)

This page is used for Guest to create a new account and become the member of Music Light. Registration page can only be accessed if the user has not logged in yet . The table below shows the list of validation for the registration page:

Field

Validation

Name

Must be filled and contain only alphabet letters.

Username

Must be filled and cannot take an existing username.

Password

Must be filled, contain numbers and alphabet letters, and more than 5 characters.

Confirm Password

Must be filled and match with password.

Gender

Must be chosen.

Address

Must be filled and contain street word.

Phone

Must be filled and contain only numbers.

Email

Must be filled and match with email format.

Example:

admin@admin.com correct

abc@ad@.com wrong

 

3.      Product page (product.php)

This page can be accessed by Guest, Member, and Admin. This page displays every available product that is sold by Music Light. Every Product must show Image, Product ID, Brand, Instrument Type, Price, and Stock. Also give paging and searching features in this page. Each page contains only 3 products and all visitors can search by Brand, Instrument Type, or Price.

Important information for this page is:

         Guest can only view the products which have Stock more than 0.

         Member can view the products which have Stock more than 0 and add the product to his/her cart. The table below shows the list of validation for the members when adding products to their cart:

Field

Validation

Quantity

Must be filled, contain only numbers, and must not bigger than the available stock.

Update quantity if the member buys the same product.

         Admin can view all, update, and delete the products.

 

4.      Insert product page (insertproduct.php)

This page can only be accessed by Admin. This page is used by Admin to insert a new product. Information that must be stored when inserting a new product is Brand, Instrument Type, Price, Stock, and Image. You are required to add upload file feature in this page to upload product image. The table below shows the list of validation for the Insert Product:

Field

Validation

Brand

Must be filled and more than 3 characters.

Instrument Type

Must be chosen.

Price

Must be filled and contain only numbers and bigger than 500000.

Stock

Must be filled and contain only numbers and bigger than 10.

Image

Must be filled and chosen from a file with .jpg extension.

 

5.      Update product page (updateproduct.php)

This page can only be accessed by Admin through Product page. This page is used by Admin to update the existing product. Information that is needed to update a Product is Brand, Instrument Type, Price, Stock, and Image. If Admin do not upload a new image, it means the product still use the old image. Validation in this page is same as Insert Product.

 

6.      Profile page (profile.php)

This page can only be accessed by Member or Admin. This page shows Username, Name, Gender, Address, Cell Phone, and Email of the user. In this page, user can change his/her profile. The table below shows the list of validation for change profile form:

Field

Validation

Name

Must be filled and contain only alphabet letters.

Address

Must be filled and contain street word.

Must contain 2 words or more.

Phone

Must be filled and contain only numbers.

Email

Must be filled and match with email format.

Example:

admin@admin.com correct

abc@ad@.com wrong

Below is the change profile form, there will be change password form for the user to change his/her password. The table below shows the list of validation for change password form:

Field

Validation

Old Password

Must be filled and match the users password in database.

New Password

Must be filled and contain numbers and alphabet letters.

Confirm Password

Must be filled and match New Password.

 

7.      Member page (member.php)

This page can only be accessed by Admin. It is used to view all information of members except the password. Furthermore, Admin can also delete member by clicking the delete bexcept him/herself. Provide paging feature in this page, each page shows only 5 members.

 

8.      Cart page (cart.php)

This page can only be accessed by Member. The function of this page is to display every product that has been added by the member from the product page. The information that must be displayed in each product which has chosen is Image, Product ID, Brand, Instrument Type, Price, Quantity, and Sub Total (Price * Quantity). Then add Total Price of all products listed in the cart page. Each product will have a Delete button that used for member to delete/cancel the product from the cart page. The last one, add Buy button to add all information listed in the cart to the Transaction page and clear the cart page. Member must wait until Admin approve the transaction.

 

9.      Transaction page (transaction.php)

This page can only be accessed by Member and Admin. This page display Transaction ID, Username, Date, and Status of the transactions. Add link to detail.php in each Transaction ID, so user can view the details about the selected transaction. Member can only view his/her transactions. Admin can view all transactions made by members and have feature for delete and approve each transaction. If Admin approves a transaction, validate if the Stock of the product is still bigger than or equal to the Quantity of product in transaction page. If the transaction is valid (Stock >= Quantity), do not forget to reduce Stock (as many as Quantity), and update the Date so it becomes the date when Admin approve the transaction. But if the transaction is not valid (Stock < Quantity) shows error message that tells users about the insufficient stock.

 

10.  Detail page (detail.php)

This page can only be accessed by Member and Admin through Transaction page. The function of this page is to display the detail about the transaction. This page only shows Image, Product ID, Brand, Instrument Type, Price, Quantity, Sub Total and Total Price of all products. Validate the Member can only view his/her transactions, while Admin can view all Members transactions.

 

11.  Testimony page (testimony.php)

This page can be accessed by Guest, Member, and Admin. It displays the testimony from Member and Admin. For Guest, he/she can only view the testimonies. For Member, he/she can view the testimonies and write (add) testimony for Music Light but it will be shown after Admin approve his/her testimony. For Admin, he/she can approve and delete the testimony from Member. Admin can also write testimony directly without someone else permission.

Field

Validation

Testimony

Must be filled.

Must be between 5 and 160 characters.

Must not contain HTML tags.

 

12.  Logout page (logout.php)

This page can only be accessed by logged in user to redirect him/her back to the home page for guest/non-member visitor and clear all sessions that have been stored.
